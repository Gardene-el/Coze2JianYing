## 📝 使用示例

### 用户视角流程

1. **访问 Issues 页面**
   ```
   https://github.com/Gardene-el/Coze2JianYing/issues
   ```

2. **点击 "New Issue" 按钮**
   
   看到 4 个模板选项：
   - 🐛 Bug 报告
   - ✨ 功能请求
   - ❓ 使用问题
   - 📚 文档改进
   
   以及 3 个外部链接：
   - 💬 讨论区
   - 📚 项目文档
   - 🔌 Coze 插件

3. **选择合适的模板**
   
   例如选择 "Bug 报告"

4. **填写表单**
   
   - 从下拉菜单选择组件
   - 在文本区域填写详细信息
   - 可选择上传截图
   - 勾选确认项

5. **提交 Issue**
   
   自动应用标签：`bug`, `需要确认`
   标题自动添加前缀：`[Bug]:`

## 🎨 模板预览示例

### Bug 报告模板填写示例

```yaml
标题: [Bug]: 草稿生成器无法处理特殊字符

问题发生在哪个组件？
└─> 草稿生成器应用 (CozeJianYingDraftGenerator.exe)

问题描述:
当 JSON 数据中包含特殊字符（如 emoji）时，
草稿生成器会崩溃并显示编码错误。

复现步骤:
1. 打开草稿生成器
2. 粘贴包含 emoji 的 JSON 数据
3. 点击"生成草稿"
4. 观察到应用崩溃

期望行为:
应该能够正确处理 emoji 和特殊 Unicode 字符

实际行为:
应用崩溃，显示 UnicodeEncodeError

运行环境:
- 操作系统：Windows 11 专业版
- 应用版本：v0.1.1
- Python 版本：3.11.5
```
